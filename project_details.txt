# Ultimate Project Documentation: Smart Archaeological Sieve & Detector

## 1. System Overview
The **Smart Archaeological Sieve & Detector** is a full-stack IoT and AI solution for archaeological excavations. It bridges high-level Vision-Language Models (VLM) and local Object Detection with low-level hardware automation via ESP32 and Firebase.

## 2. Artificial Intelligence Specifications
### Object Detection Layer
- **Core Model**: `yolov8x-worldv2.pt` (Ultralytics YOLO-World).
- **Open-Vocabulary Support**: Allows real-time detection of custom classes.
- **Pre-defined Dataset (`data.yaml`)**:
    - `0: Artifact`
    - `1: Stone`
    - `2: Glass`
    - `3: Plastic`
- **Inference Optimization**: Headless OpenCV integration using `libgl1` and `libglib2.0-0`.

### Vision-Language Integration (VLM)
- **Model**: `nvidia/nemotron-nano-12b-v2-vl:free` (NVIDIA Nemotron via OpenRouter).
- **Image Pre-processing**: Auto-resizing to **512px** for low-latency API response.
- **Bilingual Engine**: Integrated **English-to-Arabic** translation via Google Translate API for localized field reports.

## 3. Hardware & Firmware Engineering
### ESP32 Pin Mapping (Main Board)
- **Microcontroller**: ESP32 (Firmware: `esp32_firebase.ino`).
- **Load Cells (HX711 Interface)**:
    - Area 1: DOUT(16), SCK(4)
    - Area 2: DOUT(17), SCK(5)
    - Area 3: DOUT(18), SCK(19)
    - *Calibration*: Pre-set at **228.0f** for 5kg sensors.
- **Actuators (L298 Driver & MOSFETs)**:
    - Layer 1 DC Motor: Pins 12, 14 (Bidirectional: Forward/Backward/Stop)
    - Layer 2 DC Motor: Pins 27, 26 (Bidirectional: Forward/Backward/Stop)
    - Vibration Motors: Pins 32, 33, 25 (Controlled together).

### Cloud & Communication
- **Platform**: Firebase Realtime Database (RTDB).
- **Update Frequency**: Sensor weights are uploaded every **1000ms**.
- **Auth Strategy**: Anonymous signup enabled for streamlined field deployment.

## 4. Environment & Infrastructure
### Development Container (`.devcontainer`)
- **Base Image**: `python:1-3.11-bookworm`.
- **Automated Setup**: Seamless installation of `packages.txt` and `requirements.txt` via `updateContentCommand`.
- **Deployment**: Streamlit server pre-configured with disabled CORS/XSRF for easy remote access during testing.
- **Port Mapping**: Defaulting to **8501**.

### Project Architecture
- `app.py`: Central dashboard (Streamlit).
- `src/detector.py`: Dynamic model loader (YOLO vs. YOLO-World).
- `src/vlm.py`: Base64 image encoding and OpenRouter communication logic.
- `src/utils.py`: Recursive object counting and bounding box rendering.
- `src/train.py`: Local GPU/CPU training logic for custom weights.

## 5. User Interface Features
- **Dual Camera Architecture**:
    - `CV2 Webcam`: High-performance local detection.
    - `Browser Camera`: Cloud-optimized snapshot mode for mobile/tablet.
- **Interactive Stats**: Real-time object count metrics and session-based detection summaries.
- **Hardware Toggles**: Direct control over mechanical sieving layers from the web UI.

## 6. Full Dependency List
- **Python**: `ultralytics`, `streamlit`, `firebase-admin`, `opencv-python-headless`, `pillow`, `requests`.
- **System**: `libgl1`, `libglib2.0-0`.
- **Arduino**: `WiFi.h`, `Firebase_ESP_Client.h`, `HX711.h`.
